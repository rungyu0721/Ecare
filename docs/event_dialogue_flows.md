# E-CARE 事件對話流程整理

這份文件整理目前 E-CARE 聊天機器人需要支援的事件類型、應該追問的欄位、風險判斷、派遣建議，以及後續需要補強的內容。

目標是讓使用者感覺是在自然聊天，但後端仍能整理出警察端/通報端需要的結構化資料。

## 共通原則

- 先安撫，再確認安全，再追問關鍵資訊。
- 不重複問已經回答過的問題。
- 使用者說「有人、他們、對方」時，通常指現場其他人，不是使用者本人。
- 使用者說「我」時，才視為通報者本人。
- 高風險情境要優先提醒：保持距離、不要介入、移動到安全位置、通知專業人員。
- 前端可以自然聊天；後端至少要維護這些欄位：
  - `category`
  - `location`
  - `people_injured`
  - `weapon`
  - `danger_active`
  - `conscious`
  - `breathing_difficulty`
  - `fever`
  - `risk_level`
  - `dispatch_advice`

## 1. 待確認

使用者描述還不明確，例如：

- 「外面怪怪的」
- 「我聽到很大的聲音」
- 「有人在喊」
- 「我不知道發生什麼事」

應追問：

1. 現場發生什麼事？
2. 有沒有人受傷或需要幫忙？
3. 你目前在哪裡？是否安全？

建議回覆：

```text
我先幫你釐清狀況。你看到或聽到的是火災、天然災害、電梯受困、走失、衝突、車禍，還是有人身體不舒服？
```

目前需要補強：

- 判斷「喊救命」應該提高風險，不要只停在待確認。
- 判斷「很大聲響」是否接近噪音、車禍、爆炸、衝突。
- 對話流程應盡快導向明確類別。

## 2. 暴力事件

範例：

- 「有人在捷運站打架」
- 「有人拿刀追人」
- 「他們互相用拳頭互毆」
- 「有人闖進家裡」
- 「對方在威脅我」

核心欄位：

- `weapon`: 對方/現場的人是否有武器
- `people_injured`: 是否有人受傷、流血、倒地
- `danger_active`: 衝突是否還在持續、對方是否還在現場
- `location`: 位置

建議流程：

1. 提醒保持安全距離，不要靠近或介入。
2. 問是否有武器。
3. 問是否有人受傷。
4. 問衝突是否仍在持續。
5. 建議通知警方、站務人員或現場保全。

建議問句：

```text
請先保持安全距離，不要靠近衝突中的人。你有看到對方或現場的人拿刀、棍棒或其他武器嗎？
```

```text
了解，先不要靠近他們。現場有沒有人受傷或流血？
```

```text
了解。請先通知站務人員或警方。現場衝突還在持續嗎？
```

風險判斷：

- High:
  - 有武器
  - 有人受傷、流血、倒地
  - 衝突仍持續
  - 闖入、威脅、追人
- Medium:
  - 激烈爭吵、疑似要動手，但未明確動手
- Low:
  - 已停止且沒人受傷，但仍可建議通報站務/警方備查

目前已補強：

- 可以理解「沒有，他們用手互毆」代表 `weapon=false`。
- 可以理解「沒有其他人受傷」代表 `people_injured=false`。
- 已避免沒人受傷後再問呼吸、昏倒、意識。

目前需要補強：

- 「他們已經散了」「警察來了」「站務人員到了」應降低後續追問強度。
- 「我被打」「他在追我」要分辨通報者本人危險，語氣與建議要更急迫。
- 「家暴」「闖入」「跟蹤」可先歸暴力事件或可疑人士，但流程要更精準。

### 2.1 疑似家暴 / 兒少保護

這類事件常被使用者講成「隔壁很吵」「小孩一直哭」「有人在罵小孩」，模型不能只當噪音處理。

典型輸入：

- 「隔壁的小孩在哀號」
- 「樓上有大人在罵小孩，還有摔東西」
- 「隔壁小孩一直哭，有人叫他閉嘴」
- 「聽到小孩求救」
- 「隔壁像家暴」

分類建議：

- `category`: 暴力事件
- `risk_level`: Medium 到 High
- 有求救、打罵、摔東西、武器、持續哭叫時，偏 High
- `dispatch_advice`: 建議派遣警察，必要時救護車待命

回覆原則：

1. 不要問「你最擔心哪個小孩？」這類一般聊天問題。
2. 不要建議使用者自行敲門或介入。
3. 先提醒保持距離與自身安全。
4. 問聲音是否仍持續、是否有求救/打罵/摔東西。
5. 若有武器或明顯受傷，立即建議通知警察並說明兒少疑似受害。

建議回覆：

> 這可能涉及家暴或兒少安全，請先不要靠近或敲門。你現在還聽得到小孩哀號、哭叫或摔東西的聲音嗎？

若使用者說「有人持刀」：

> 這是高風險情況，請不要靠近或介入。請在安全位置立即通知警察或管理員，並告知疑似兒少受害、現場有人持有武器。

## 3. 醫療急症

範例：

- 「有人昏倒」
- 「他沒有反應」
- 「朋友呼吸很困難」
- 「有人流很多血」
- 「我胸口很痛」
- 「小孩發燒很高」

核心欄位：

- `reporter_role`: 是本人，還是替他人通報
- `conscious`: 是否有意識、叫得醒
- `breathing_difficulty`: 是否呼吸困難、沒呼吸、喘不過氣
- `people_injured`: 是否有人受傷
- `symptom_summary`: 症狀摘要
- `location`

建議流程：

1. 判斷是否本人或他人。
2. 問意識是否清楚。
3. 問呼吸是否正常。
4. 問是否大量出血、胸痛、抽搐、昏倒。
5. 高風險時建議立刻撥打 119，尋找 AED，等待救護人員。

建議問句：

```text
請先確認他是否有反應、叫得醒嗎？
```

```text
他目前呼吸是否正常？有沒有喘不過氣、嘴唇發紫或沒有呼吸？
```

```text
如果沒有反應或呼吸異常，請立刻撥打 119，並請旁邊的人協助找 AED。
```

風險判斷：

- High:
  - 沒反應、叫不醒
  - 沒呼吸、呼吸困難
  - 大量出血
  - 胸痛、抽搐、嚴重過敏
- Medium:
  - 發燒高、摔倒疑似骨折、症狀加重
- Low:
  - 輕微擦傷、可自行移動且狀況穩定

目前需要補強：

- 區分「本人不舒服」與「看到別人不舒服」。
- 若使用者已回答呼吸正常，不要重複問呼吸。
- 若事件原本是暴力/車禍且已確認沒受傷，不要跳入醫療流程。
- AED、CPR 的建議要更謹慎：不要要求未受訓者進行複雜處置。

### 3.1 偏鄉 / 山區 / 國家公園救援

新定位下，E-CARE 主要支援偏鄉、山區與國家公園使用者。這類狀況不一定一開始就像醫療急症，系統要先判斷需要 119、110，或兩者都需要。迷路、受困、失聯、摔落、無法行走、溪水暴漲、失溫、中暑、高山症、溺水、落石坍方、手機快沒電 / 沒訊號通常導向 119；持刀、跟蹤、闖入、暴力、犯罪或人身威脅則導向 110；若同時有人受傷、火災或受困，提醒同步 119。

典型輸入：

- 「我們在國家公園步道迷路了」
- 「朋友在山上摔落，現在不能走」
- 「溯溪時溪水暴漲，我們被困住」
- 「山上有人失溫，手機快沒電」
- 「偏鄉家人突然不舒服，但救護車不知道怎麼找到」

核心欄位：

- `location`: GPS 座標、步道名稱、登山口、里程牌、山屋、明顯地標
- `people_injured`: 是否有人受傷、失溫、中暑、溺水、無法行走
- `danger_active`: 是否仍迷路、受困、溪水暴漲、落石坍方、天色變暗
- `symptom_summary`: 山域水域救援、傷勢或環境風險摘要
- `dispatch_advice`: 建議通報 119，提供座標、地標、同行人數與傷勢

建議流程：

1. 若有立即危險，先判斷需要 119、110，或兩者都需要。
2. 優先確認 GPS 座標或最近地標 / 步道名稱。
3. 問同行人數、是否有人受傷或無法移動。
4. 問手機電量、訊號、天候與天色。
5. 提醒待在相對安全處，不要冒險下切溪谷、靠近崖邊或自行搬動疑似骨折/墜落傷者。
6. 不假設山區或偏鄉附近有 AED；除非使用者明確說現場有 AED，否則不要主動要求尋找 AED。

建議問句：

```text
這屬於需要 119 協助的山域救援情境。請先停在安全位置，不要下切溪谷或靠近崖邊；你現在有 GPS 座標、步道名稱或附近地標嗎？
```

```text
你們同行幾個人？目前有人受傷、失溫、中暑，或無法自行行走嗎？
```

風險判斷：

- High:
  - 迷路且手機快沒電 / 沒訊號
  - 受困、失聯、溪水暴漲、落石坍方
  - 摔落、疑似骨折、無法行走、溺水、失溫、高山症
  - 山屋、步道、偏鄉住家或停車場出現持刀、跟蹤、闖入、暴力或人身威脅
- Medium:
  - 在偏遠地區身體不適、天色變暗、交通或定位不明
- Low:
  - 已脫困、已下山、救援已到且無傷勢

## 4. 火災

範例：

- 「廚房起火」
- 「樓上冒濃煙」
- 「聞到焦味」
- 「電線冒煙」
- 「有人受困在裡面」

核心欄位：

- `danger_active`: 火勢/濃煙是否仍在持續
- `people_injured`: 是否有人受困、受傷、吸入濃煙
- `location`: 起火位置
- `symptom_summary`: 是否嗆傷、燙傷

建議流程：

1. 先提醒離開火場，不要回頭拿東西。
2. 問火勢或濃煙是否仍持續。
3. 問是否有人受困或受傷。
4. 問起火點大概在哪裡。
5. 建議撥打 119，關門阻隔濃煙，勿搭電梯。

建議問句：

```text
請先離開火場並保持低姿勢避開濃煙，不要搭電梯。現場火勢或濃煙還在持續嗎？
```

```text
現場有沒有人受困、受傷，或吸入濃煙不舒服？
```

風險判斷：

- High:
  - 明火、濃煙、有人受困、火勢蔓延
- Medium:
  - 焦味、冒煙但未確認明火
- Low:
  - 疑似異味，尚未確認危險

目前需要補強：

- 區分「看到火」與「聞到焦味」。
- 增加「瓦斯味、電線走火、電器冒煙」。
- 地址/樓層/起火點應優先追問。

## 5. 交通事故

範例：

- 「發生車禍」
- 「機車騎士摔倒」
- 「有人被困在車內」
- 「路中間有事故」
- 「有人流血」

核心欄位：

- `people_injured`: 是否有人受傷、受困、流血
- `danger_active`: 車輛是否仍在車道、是否有二次事故風險
- `location`
- `symptom_summary`: 傷勢

建議流程：

1. 提醒到安全位置，不要站在車道中。
2. 問是否有人受傷或受困。
3. 問車輛是否阻擋道路、是否有漏油/起火。
4. 建議報警與叫救護車。

建議問句：

```text
請先移到安全位置，不要站在車道上。現場有沒有人受傷或被困住？
```

```text
車輛是否還在車道中，或有漏油、冒煙、起火的情況？
```

風險判斷：

- High:
  - 有人受困、嚴重受傷、大量出血、車輛起火/漏油
- Medium:
  - 有人摔倒、疑似受傷、阻礙交通
- Low:
  - 輕微擦撞且無人受傷

目前需要補強：

- 「無人受傷但車在路中」仍有交通風險。
- 「已報警」「救護車已到」應改成等待與保護現場。
- 增加行人、機車、汽車、校園/路口等細分情境。

## 6. 可疑人士

範例：

- 「有人一直跟著我」
- 「陌生人在社區徘徊」
- 「有人在門口不走」
- 「有人一直看我家」

核心欄位：

- `danger_active`: 對方是否仍在附近、是否持續跟隨
- `weapon`: 是否有武器或威脅行為
- `location`
- `reporter_role`: 是否本人被跟蹤

建議流程：

1. 提醒不要直接接觸對方。
2. 問對方是否仍在附近或跟著。
3. 問是否有威脅、武器、試圖靠近。
4. 建議去人多/明亮處，通知警方、警衛或店家。

建議問句：

```text
請先往人多或明亮的地方移動，不要直接接觸對方。對方現在還在附近或還跟著你嗎？
```

```text
你有看到對方拿武器，或有威脅、靠近、闖入的行為嗎？
```

風險判斷：

- High:
  - 持續跟蹤、靠近、威脅、闖入、有武器
- Medium:
  - 長時間徘徊、可疑但未直接威脅
- Low:
  - 單次可疑觀察，無持續危險

目前需要補強：

- 區分「我被跟蹤」與「看到陌生人在社區」。
- 若使用者本人正在被跟蹤，應優先引導到安全地點。
- 家門口/住家附近情境要更謹慎。

## 7. 噪音

範例：

- 「隔壁很吵」
- 「樓上一直大吼」
- 「深夜施工」
- 「有人一直咆哮」

核心欄位：

- `danger_active`: 噪音是否仍持續
- `people_injured`: 是否伴隨受傷、求救、哭喊
- `weapon`: 是否有威脅或暴力跡象
- `location`

建議流程：

1. 先確認是否只是噪音，或已經有衝突/求救。
2. 問是否聽到威脅、打鬥、求救、東西破裂。
3. 若只是噪音，建議記錄時間、通知管理員或警方。
4. 若有暴力跡象，轉成暴力事件。

建議問句：

```text
我先幫你確認安全性。你有聽到求救、打鬥聲、威脅，或東西被砸的聲音嗎？
```

```text
如果只是持續噪音，請記錄時間與位置，必要時通知管理員或警方協助。
```

風險判斷：

- High:
  - 伴隨求救、打鬥、暴力、威脅
- Medium:
  - 激烈爭吵、咆哮、砸東西
- Low:
  - 單純施工、音樂、吵鬧

目前需要補強：

- 「大吼大叫」不一定是噪音，可能是家暴或衝突。
- 需要明確轉類別：噪音 -> 暴力事件。
- 增加夜間、社區、校園、宿舍情境。

## 8. 天然災害

範例：

- 「地震後大樓倒塌」
- 「豪雨淹水，一樓有人受困」
- 「颱風停電，附近道路中斷」
- 「土石流把路沖斷」

核心欄位：

- `danger_active`: 餘震、淹水、土石流、倒塌、道路中斷等危險是否仍持續
- `people_injured`: 是否有人受困、被壓住、受傷或需要救護
- `location`: 災害地點、樓層、道路、橋梁、山坡地或溪流位置

建議流程：

1. 先提醒遠離倒塌、淹水、土石流、瓦斯味等危險區域。
2. 問災害危險是否仍在持續。
3. 問是否有人受困、被壓住或受傷。
4. 建議優先撥打 119，並準備告知災害類型、位置、受困/受傷狀況。

建議問句：

```text
請先遠離倒塌、淹水、土石流或瓦斯味等危險區域。現場危險還在持續嗎？
```

```text
現場有沒有人受困、被壓住、受傷，或需要救護車？
```

風險判斷：

- High:
  - 地震、建築物倒塌、道路中斷、土石流、有人受困、被壓住、有人被埋
- Medium:
  - 颱風、豪雨、淹水、積水、停電、餘震但未明確受困受傷
- Low:
  - 已撤離、消防到了、目前安全且無人受傷

## 9. 受困救援

範例：

- 「有人電梯受困」
- 「我們困在電梯，門打不開」
- 「電梯卡住，有人開始不舒服」

核心欄位：

- `danger_active`: 是否仍受困，門是否仍打不開
- `people_injured`: 是否有人受傷、不舒服、呼吸困難，是否有老人、小孩、孕婦
- `location`: 地址、樓層、電梯編號或明顯地標

建議流程：

1. 先提醒不要強行開門、攀爬或自行脫困。
2. 問是否仍受困。
3. 問受困人數與是否有人不適。
4. 建議撥打 119 或請管理員同步通知消防。

建議問句：

```text
請不要強行開電梯門或攀爬。人現在還困在電梯裡嗎？
```

```text
電梯裡有幾個人？有沒有人受傷、不舒服、呼吸困難，或有老人小孩孕婦？
```

風險判斷：

- High:
  - 電梯受困仍持續、門打不開、有人不舒服或呼吸困難
- Medium:
  - 電梯異常但未明確有人不適
- Low:
  - 已脫困且無人不適

## 10. 自殺危機

範例：

- 「有人站在頂樓說要跳樓」
- 「他說想自殺」
- 「朋友割腕了」
- 「她吞藥說不想活」

核心欄位：

- `danger_active`: 是否仍在頂樓、陽台邊、持刀、已吞藥、燒炭或一個人
- `people_injured`: 是否已受傷、流血、吞藥、昏倒或沒反應
- `location`: 精確位置、樓層、房號或現場地標

建議流程：

1. 不刺激、不責備、不要求通報者拉扯或自行制服對方。
2. 提醒保持安全距離、陪伴並移除可安全移除的危險物。
3. 問是否仍在危險位置或是否已有自傷行為。
4. 建議同步撥打 119 與 110。

建議問句：

```text
請先不要刺激或拉扯對方，保持陪伴與安全距離。對方現在還在頂樓、陽台邊、持刀或已經吞藥嗎？
```

```text
對方目前有受傷、流血、吞藥、昏倒或沒有反應嗎？
```

風險判斷：

- High:
  - 跳樓、自殺、割腕、吞藥、燒炭、上吊、站在危險位置
  - 即使已有人陪伴，也仍維持 High，直到專業協助介入或明確安全
- Low/緩和:
  - 已離開危險位置、有人陪伴、警消或救護到場；`danger_active` 可為 false，但仍需專業協助

## 11. 失蹤走失

範例：

- 「小孩在市區公園走失」
- 「老人走失找不到」
- 「家人失聯，電話打不通」
- 「國家公園裡有人失聯，手機沒訊號」

核心欄位：

- `danger_active`: 是否仍找不到、失聯或手機關機
- `people_injured`: 是否為小孩、長輩、失智、身心障礙、需服藥或可能受困受傷
- `location`: 最後出現時間、地點、穿著、照片、可能移動方向

建議流程：

1. 區分一般市區走失與山區/水域/偏鄉失聯。
2. 一般走失優先 110；山區、水域、偏鄉、受困或受傷風險同步 119。
3. 問最後出現時間、地點、穿著、照片、聯絡方式。
4. 問是否為高風險對象：小孩、老人、失智、需要服藥。

建議問句：

```text
請先確認最後看到人的時間、地點與穿著。現在還聯絡不上或找不到人嗎？
```

```text
走失的人是小孩、長輩、失智或需要服藥的人嗎？有沒有可能受傷或受困？
```

風險判斷：

- High:
  - 小孩失蹤、老人/失智走失、夜間走失、手機關機、疑似被帶走
  - 山區/國家公園/水域失聯且沒訊號或手機低電量，應轉 `山域水域救援`
- Medium:
  - 一般失聯或走失，但尚未出現立即危險線索
- Low:
  - 已找到、已聯絡上、警方到場且人員安全

## 建議優先補強順序

1. 維持自殺危機、OHCA、中風、心肌梗塞等高風險字面詞的高召回。
2. 持續補混淆測試：市區走失 vs 山區失聯、電梯受困 vs 火災、噪音 vs 家暴/暴力。
3. 擴充天然災害與受困救援的多輪短答：已脫困、消防到了、沒有受傷、仍受困。
4. 補更多否定句與緩和狀態：沒受傷、沒有武器、已離開危險位置、已聯絡上。
5. 待確認流程需要快速導向上述明確類別。

## 訓練資料需要增加的內容

每個事件至少補以下類型：

- 使用者第一句簡短模糊描述
- 使用者回答「沒有 / 有 / 不確定」
- 使用者用代名詞回答：「他們」「那個人」「對方」「我朋友」
- 使用者回答已處理：「警察來了」「站務人員到了」「已經散了」
- 使用者情緒強烈：「我很怕」「快點」「怎麼辦」
- 低風險收尾：「已經沒事了」「沒有人受傷」

建議每類至少補 30 到 50 筆多輪對話資料。優先補自殺危機、失蹤走失、受困救援、天然災害，以及暴力/醫療/火災/交通事故的混合情境。

## 台灣事件分類擴充參考

這份分類用來輔助 E-CARE 判斷使用者描述屬於哪一種事件，並給出對應的第一步建議。實作上仍以「是否有立即人身危險」優先，避免把非緊急法律或醫療糾紛全部升級成 110/119。

| 大類 | 常見子類 | 優先建議 |
| --- | --- | --- |
| 刑事案件 | 暴力犯罪、竊盜、詐欺、毒品、家暴、性侵、網路犯罪、特殊刑案、失蹤人口/走失 | 暴力、家暴、性侵、特殊刑案與失蹤走失優先 110；詐欺提醒 165；家暴、兒少、性侵可提醒 113 |
| 災害救護事件 | 火災、天然災害、受困救援、緊急救護、自殺危機、交通事故、山域/水域救援、危險物品事故 | 優先 119；自殺危機同步 110；交通事故若需交通管制或現場衝突，可同步 110 |
| 醫療事件 | 重大醫療事故、病人安全事件、醫療糾紛 | 若有立即生命危險先 119；若為醫療疏失或糾紛，先確保病人安全並保存紀錄 |
| 民事/社會事件 | 民事訴訟、家事事件、少年事件、行政訴訟 | 通常不是緊急報案；保存證據並洽調解、法律諮詢、法院或行政機關 |

判斷原則：
- 先問是否有人正在受傷、受困、遭威脅、呼吸困難、意識不清，或現場危險是否仍持續。
- 有立即危險時，安全、撤離、110/119 分流優先：醫療、消防、天然災害、受困救援、山域水域救援偏 119；自殺危機同步 119/110；失蹤走失、暴力、犯罪、人身威脅偏 110；混合情境同步提醒。
- 無立即危險時，回覆應轉為保存證據、紀錄時間地點、提供對應單位，不要硬推緊急派遣。
